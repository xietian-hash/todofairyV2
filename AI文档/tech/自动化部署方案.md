# 自动化部署方案

## 1. 目标

- 当代码推送到 `main` 分支时，自动部署后端服务
- 部署目标服务器为 Ubuntu 24.04
- 部署目录固定为 `~/api-todofairyV2`
- 部署方式使用 `docker compose`
- 生产环境变量通过 GitHub Secrets 下发

## 2. 部署流程

1. GitHub Actions 监听 `main` 分支推送
2. Workflow 打包仓库部署所需文件，排除 `.git`、`node_modules`、`data`、`历史数据`、`密钥`
3. Workflow 通过 SSH/SCP 将部署包上传到服务器 `~/api-todofairyV2`
4. Workflow 在服务器写入 `.env.production`
5. Workflow 在服务器执行 `docker compose -f docker-compose.deploy.yml build --pull`
6. Workflow 在服务器执行 `docker compose -f docker-compose.deploy.yml run --rm api pnpm --filter @todo-fairy/api prisma:migrate:deploy`
7. Workflow 在服务器执行 `docker compose -f docker-compose.deploy.yml up -d api`

## 3. 仓库内文件

- Workflow：`.github/workflows/deploy-api.yml`
- 后端镜像：`apps/api/Dockerfile`
- 部署编排：`docker-compose.deploy.yml`
- 镜像构建忽略：`.dockerignore`

## 4. GitHub Secrets

- `DEPLOY_HOST`：服务器 IP 或域名
- `DEPLOY_PORT`：SSH 端口
- `DEPLOY_USER`：SSH 用户名
- `DEPLOY_SSH_KEY`：SSH 私钥内容
- `PROD_ENV_FILE`：生产环境 `.env.production` 完整文本

## 5. 运行时环境变量

`PROD_ENV_FILE` 至少应包含以下变量：

```env
PORT=3000
DATABASE_URL=mysql://user:password@127.0.0.1:3306/tasksprite
JWT_SECRET=replace_me
JWT_EXPIRES_IN=7d
WECHAT_APPID=replace_me
WECHAT_SECRET=replace_me
INTERNAL_API_KEY=replace_me
NODE_ENV=production
```

## 6. 迁移约束

- 生产环境只允许执行 `prisma migrate deploy`
- 不允许在生产环境使用 `prisma db push`
- 必须先提交 Prisma migration 文件，再允许进入自动部署流程

## 7. 风险与约束

- `main` 分支的每次推送都会触发部署，因此 `main` 必须保持可发布状态
- 服务器需预先安装 Docker 与 Docker Compose 插件
- 服务器 SSH 用户需对部署目录和 Docker 具备执行权限
- 数据库变更失败会导致部署中断，避免未验证的 migration 直接进入 `main`
