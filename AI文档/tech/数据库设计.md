# 数据库设计

## 1. 数据库

- 数据库类型：MySQL 8
- 默认数据库名：`tasksprite`
- ORM：Prisma
- Schema 文件：`apps/api/prisma/schema.prisma`

## 2. 核心表

### 2.1 `user`

- `id`
- `nickname`
- `avatar_url`
- `status`
- `last_login_at`
- `created_at`
- `updated_at`

### 2.2 `user_identity`

- `id`
- `user_id`
- `provider`
- `identity_key`
- `openid`
- `unionid`
- `created_at`
- `updated_at`

说明：
- 当前小程序登录使用 `provider=wechat_miniprogram`
- `(provider, identity_key)` 唯一

### 2.3 `user_credential`

- `id`
- `user_id`
- `password_hash`
- `password_algo`
- `password_updated_at`
- `failed_login_count`
- `lock_until`
- `created_at`
- `updated_at`

说明：
- 当前版本主要走微信登录
- 该表为后续账号密码扩展预留

### 2.4 `user_tag`

- `id`
- `user_id`
- `name`
- `color`
- `sort`
- `is_deleted`
- `created_at`
- `updated_at`
- `deleted_at`

### 2.5 `task`

- `id`
- `user_id`
- `title`
- `remark`
- `tag_id`
- `tag_name`
- `effective_start_date`
- `effective_end_date`
- `repeat_rule_json`
- `status`
- `is_deleted`
- `version`
- `sub_task_enabled`
- `sub_tasks_json`
- `deleted_at`
- `created_at`
- `updated_at`

说明：
- `repeat_rule_json` 保存重复规则
- `sub_tasks_json` 保存子任务配置

### 2.6 `todo`

- `id`
- `user_id`
- `task_id`
- `parent_task_id`
- `parent_todo_id`
- `is_sub_todo`
- `sub_task_index`
- `sub_task_title`
- `task_version`
- `todo_date`
- `trigger_type`
- `title`
- `remark`
- `tag_id`
- `tag_name`
- `status`
- `completed_at`
- `is_expired`
- `expired_at`
- `is_deleted`
- `deleted_at`
- `created_at`
- `updated_at`

说明：
- 父待办使用真实 `task_id`
- 子待办沿用业务语义，`task_id` 允许使用派生值，不对 `task.id` 建外键
- `(user_id, task_id, todo_date)` 唯一，保证同一待办生成幂等

### 2.7 `todo_generation_log`

- `id`
- `user_id`
- `task_id`
- `todo_date`
- `trigger_type`
- `result`
- `todo_id`
- `error_code`
- `error_message`
- `trace_id`
- `created_at`

说明：
- 用于记录待办生成结果
- `result` 当前包含：
  - `created`
  - `already_exists`
  - `failed`

### 2.8 `notification_setting`

- `id`
- `user_id`
- `send_key`
- `daily_enabled`
- `weekly_enabled`
- `daily_time`
- `weekly_time`
- `last_test_at`
- `last_test_status`
- `last_test_error_message`
- `created_at`
- `updated_at`

### 2.9 `notification_log`

- `id`
- `user_id`
- `summary_date`
- `channel`
- `status`
- `title`
- `content`
- `source`
- `error_code`
- `error_message`
- `trace_id`
- `attempt_count`
- `created_at`
- `updated_at`
- `delivered_at`
- `last_attempt_at`

说明：
- `(user_id, summary_date, channel)` 唯一
- 用于保证日报、周报发送幂等

## 3. 关键索引

- `user_identity(provider, identity_key)` 唯一
- `user_tag(user_id, name)` 唯一
- `task(user_id, is_deleted, status)`
- `task(user_id, effective_start_date, effective_end_date)`
- `todo(user_id, task_id, todo_date)` 唯一
- `todo(user_id, todo_date, status)`
- `todo(user_id, parent_task_id)`
- `todo(user_id, parent_todo_id)`
- `todo_generation_log(user_id, todo_date)`
- `todo_generation_log(task_id, todo_date)`
- `todo_generation_log(user_id, trigger_type, created_at)`
- `notification_setting(user_id)` 唯一
- `notification_log(user_id, summary_date, channel)` 唯一
- `notification_log(user_id, created_at)`

## 4. 当前实现说明

- Prisma 已接入并用于正式读写
- 已建立基线 migration：`apps/api/prisma/migrations/20260501103500_init_mysql_schema`
- 本地查看迁移状态：
  - `pnpm run prisma:migrate:status`
- 开发新增字段推荐使用：
  - `pnpm run prisma:migrate:dev -- --name <migration_name>`
- 部署环境执行：
  - `pnpm run prisma:migrate:deploy`
- `prisma db push` 仅适合快速同步或原型阶段，不应替代正式 migration 流程
