# 任务精灵重构版 PRD

## 1. 背景

现有项目需要从“微信云函数承载后端逻辑”的模式迁移为“独立 Node.js 后端 + 原生微信小程序前端”的模式，以支持统一鉴权、数据库演进、日志追踪和可部署性。

## 2. 本期目标

- 保持任务精灵现有核心功能和统计口径不回退
- 小程序请求层改为 HTTP API
- 后端统一使用 `NestJS + Prisma + MySQL`
- 提供统一的 `code / message / data / traceId` 响应结构

## 3. 本期范围

### 3.1 后端

- 微信登录
- 标签管理
- 任务管理
- 待办管理
- 月历统计
- 通知设置、测试发送、内容预览
- AI 子任务拆解
- 内部定时接口

### 3.2 前端

- 保持现有页面结构、交互和文案
- 将 `wx.cloud.callFunction` 替换为 HTTP API
- 保持首页、标签页、提醒页、菜单页可用

## 4. 关键业务约束

- 父任务统计口径不回退
- 周报自动发送仅允许周一执行
- 一次性任务生成当天待办后保持自动归档语义
- 子待办联动父待办状态

## 5. 2026-05-01 实现状态

- 后端数据层已完成从本地 JSON 文件存储迁移到 MySQL
- `auth / tags / tasks / todos / calendar / notifications / internal` 已改为 Prisma 实现
- 开发态仍支持 `devOpenId` 和 `dev:openid` 登录方式，保证本地联调效率

## 6. 验收口径

- 小程序首页、标签页、提醒页、菜单页功能可用
- 后端接口统一返回 `code / message / data / traceId`
- 所有用户资源访问经过 JWT 和 `userId` 归属校验
- 关键链路可完成：登录、建标签、建任务、生成父子待办、月历统计、通知设置、内部初始化
