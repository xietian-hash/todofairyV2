# 迁移计划

## 1. 迁移原则

- 原仓库不直接修改
- 新代码全部落在 `优化项目`
- 先迁功能，再收敛结构
- 先保行为一致，再做实现优化

## 2. 阶段划分

### 阶段一：基础骨架

- 新文档
- Monorepo 基础结构
- NestJS 后端脚手架
- 小程序代码副本迁移
- 文件存储适配层

### 阶段二：基础能力迁移

- 微信登录
- JWT
- traceId
- 错误码
- 日志
- DTO 校验

### 阶段三：核心业务迁移

- 标签
- 任务
- 待办
- 月历

### 阶段四：增强能力迁移

- 通知
- 周报
- AI 子任务
- 内部定时接口

### 阶段五：联调与验收

- 小程序请求层切换
- 页面联调
- 自测和回归

## 3. 风险项

- 微信登录 code 换 openid 需要替代云函数上下文
- 云数据库查询语义迁移到 MySQL 可能出现统计偏差
- 父任务口径、子任务联动、周报限制是高风险规则
- 数据迁移脚本需要单独验证，不能与业务迁移混做
